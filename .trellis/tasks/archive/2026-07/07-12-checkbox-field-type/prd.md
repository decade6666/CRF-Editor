# 新增字段类型「复选」

## Goal

为 CRF 编辑器增加无需字典支持的单一复选框字段类型「复选」，使用户能表达如“已签署”“已确认”等单项勾选场景，并在字段库、表单设计、预览、Word 导出及项目复制/导入中保持一致。

## Confirmed Requirements

### R1: 字段类型与入口

- 新增字段类型字符串 `复选`。
- `FieldsTab.vue` 和 `FormDesignerTab.vue` 的新建/编辑类型下拉均可选择该类型；由字段库添加至表单与设计器直接新建的行为一致。
- Word 预览双击打开的字段属性，以及其他已覆盖的字段类型展示位置，均显示 `复选`。

### R2: 虚拟单选项

- 复选字段在概念上只有一个虚拟选项：选项 OID 固定为 `1`，不提供编辑入口。
- 该选项不创建、关联或展示为 `CodeList` / `CodeListOption`；字段 `codelist_id` 必须保持 `null`，字典库不得新增条目。
- 复选不能加入 `isChoiceField()`，从而不进入现有选择类字典校验及选项编辑流程。

### R3: 复选文本

- 在字段定义级新增可空 `checkbox_label`，所有引用同一字段定义的表单共享该配置。
- 复选文本允许自定义；`checkbox_label` 为 `null` 或空字符串时，动态回退到字段定义的 `label`。
- 属性编辑器仅对 `复选` 显示“复选文本”输入框，placeholder 使用字段标签；切换离开复选类型时清除该专属配置。
- 不提供表单实例级 `label_override` 对复选文本的单独覆盖。

### R4: 渲染与布局

- 复选保持常规字段的两列布局：左单元格为字段标签，右单元格为 `□{复选文本}`。
- 前端所有预览路径与 eCRF Word 导出都必须使用同一回退规则，正确显示 `□{checkbox_label 或 label}`，不得回退为文本字段填写线。
- 复选是普通字段而非结构字段；aCRF 继续仅显示字段级变量名，不额外显示虚拟选项 OID `1`。

### R5: 数据兼容与列宽

- 在 `field_definition` 轻量迁移中添加 `checkbox_label`，对既有数据库兼容。
- 字段创建、更新、响应、项目复制、项目/模板数据库导入及模板预览均保真传递 `checkbox_label`。
- `□ + 复选文本` 的实际宽度进入前后端共享列宽规划；修改共享 fixture 生成器后，后端与前端 planner 用例同步通过。

### R6: Word 导入范围

- 本期不自动从 `.docx` 检测/推断“复选”类型；手动设置类型后，类型数据必须能正常保存、预览与导出。

## Acceptance Criteria

- [ ] AC1：字段库和表单设计器均能创建/编辑 `复选` 字段，且保存后 `codelist_id` 为 `null`。
- [ ] AC2：复选字段属性中可配置“复选文本”；为空时，前端预览与 Word 导出显示 `□{字段标签}`；填写后显示 `□{自定义文本}`。
- [ ] AC3：复选不显示字典、单位、日期格式、数值位数及默认值控件，也不触发“必须选择选项字典”校验。
- [ ] AC4：常规预览维持“字段标签 | □复选文本”两列，双击字段属性显示类型“复选”。
- [ ] AC5：字段库、字典库没有因复选创建产生的字典或选项；虚拟选项 OID 仅为固定内部语义 `1`。
- [ ] AC6：eCRF Word 导出正确显示方框及文本；aCRF 不额外产生选项 OID 批注。
- [ ] AC7：项目复制、项目数据库导入和模板导入后保留 `field_type=复选`、`checkbox_label` 与空 `codelist_id`。
- [ ] AC8：复选的列宽按 `□ + 复选文本` 参与共享 planner，前后端 fixture 契约一致。
- [ ] AC9：相关后端、前端、构建/lint 与端到端验证通过。

## Out of Scope

- 从 Word 文档自动识别“复选”字段。
- 一个复选字段承载多个选项。
- 可编辑的虚拟选项 OID 或 aCRF 选项级 OID 批注。
- 表单实例级复选文本覆盖。
