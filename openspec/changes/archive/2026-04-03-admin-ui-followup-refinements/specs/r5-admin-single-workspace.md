# Spec: R5 — 管理员界面仅保留用户管理工作区

## Scope
前端：AdminView 信息架构。
后端：继续复用 `GET /api/auth/me` 与用户管理接口。

## Functional Requirements
- 管理员界面只有一个顶层工作区：用户管理。
- 顶层不再展示历史杂项 tab 或并列管理面板。
- 用户管理最小闭环必须保留：列表、创建、改名、删除限制。
- 项目批处理与回收站如需保留，必须作为用户管理工作区内部动作出现，而非独立顶层区域。

## Acceptance Criteria
- [ ] 管理员页只有用户管理一个顶层工作区
- [ ] 用户列表、创建、改名、删除限制流程可用
- [ ] 旧顶层功能入口被移除或内聚到用户管理工作区内部
- [ ] 不存在死链入口
