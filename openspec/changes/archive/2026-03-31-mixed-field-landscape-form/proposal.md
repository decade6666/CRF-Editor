# Proposal: 混合字段横向表单统一渲染

## 背景

当前 Word 导出和前端预览中，表单字段按 `inline_mark` 分为两类独立渲染：
- **普通字段** (`inline_mark=0`): 2 列表格（标签 | 值）
- **表格字段** (`inline_mark=1`): N 列横向表格（表头 + 数据行）

当一个表单同时包含两种字段类型时，它们被拆分为多个独立表格。用户需要将所有内容统一到**同一个表格**内，并以**横向（landscape）** 方向展示，同时在该表单前插入**分节符（下一页）**。

## 需求摘要

| 维度 | 描述 |
|------|------|
| 触发条件 | 表单同时包含 inline 字段和非 inline 字段，且 inline 字段数 > 4 |
| 布局规则 | 所有字段统一渲染在一个 N 列表格中 |
| 普通字段布局 | 左侧合并单元格显示标签，右侧合并单元格显示填写区域 |
| 标签/日志行 | 全行合并为单个单元格 |
| 表格字段布局 | 保持现有的 N 列表头 + 数据行格式 |
| 页面方向 | 横向（landscape），Cm(29.7) x Cm(21) |
| 分节符 | 横向表单前插入 `WD_SECTION.NEW_PAGE`，横向表单后恢复 `PORTRAIT` |
| 数据模型 | 无需新增数据库字段（自动检测） |

## 约束集

### 硬约束

1. **列数一致性**: 统一表格的列数 = inline 字段数，普通字段通过合并单元格适配
2. **分节符必须**: 横向页面前必须有 `WD_SECTION.NEW_PAGE`，否则 Word 无法切换页面方向
3. **方向恢复**: 横向内容后必须恢复 `PORTRAIT`，避免影响后续表单
4. **表格边框**: 统一使用 `_apply_grid_table_style` 网格边框
5. **字体一致性**: 所有 run 必须通过 `_set_run_font` 设置，保持宋体/TNR 一致

### 软约束

1. **列宽分配**: 横向可用宽度 Cm(23.36)，均分给 N 列
2. **普通字段合并比例**: 建议左标签占 ~40% 列数，右值占 ~60% 列数
3. **渲染顺序**: 按 `sort_order` 保持字段原始顺序，不重排

### 依赖

1. `field_rendering.py` 的 `build_inline_table_model` 和 `extract_default_lines` 函数
2. `_add_inline_table` 和 `_build_form_table` 的现有渲染逻辑（部分复用）
3. 前端 `renderGroups` 和 `needsLandscape` 计算属性

### 风险

1. **列数过少时合并不自然**: 如 inline 只有 5 列，普通字段左标签只能占 2 列，可能显示拥挤
2. **底纹/文字颜色**: 合并单元格的颜色应用需测试
3. **纵向选项**: `_render_vertical_choices` 在合并单元格中的行为需验证

## 影响范围

### 后端

| 文件 | 变更类型 | 说明 |
|------|----------|------|
| `export_service.py` | 修改 | `_add_forms_content` 检测混合场景；新增 `_build_unified_table` 方法 |
| `field_rendering.py` | 无变更 | 现有函数可直接复用 |
| `database.py` | 无变更 | 无需迁移 |
| `models/form.py` | 无变更 | 无需新增字段 |

### 前端

| 文件 | 变更类型 | 说明 |
|------|----------|------|
| `FormDesignerTab.vue` | 修改 | `renderGroups` 检测混合场景时合并为统一组；更新预览渲染逻辑 |
| `main.css` | 可能微调 | landscape 样式可能需适配统一表格预览 |

## 成功判据

1. 混合字段表单导出的 Word 文档：所有字段在同一个 `<w:tbl>` 内
2. 该表格前有 `WD_SECTION.NEW_PAGE` 分节符
3. 该表格所在 section 为 `LANDSCAPE` 方向
4. 后续表单恢复 `PORTRAIT` 方向
5. 普通字段通过合并单元格保持标签+值的两栏布局
6. 标签/日志行全宽合并
7. 表格字段保持 N 列表头+数据行格式
8. 现有纯 inline 或纯 normal 表单的导出行为不变（回归安全）
9. 前端预览与导出行为一致
