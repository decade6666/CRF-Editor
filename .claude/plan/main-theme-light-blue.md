# 主页面主题色替换为浅蓝色

## 任务归类
- 前端样式调整
- 范围：仅主页面现有主题色，从紫色切换为浅蓝色
- 原则：最小改动，不调整布局、不改组件结构、不扩大到无关功能

## 增强后的需求
### 目标
将主页面当前以紫色/靛蓝色为主的视觉主题改为浅蓝色系，并保持现有界面层级、交互状态与可读性。

### 技术约束
- 继续沿用现有 CSS 变量体系
- 继续沿用 Element Plus 主题变量覆盖机制
- 优先通过单点主题变量替换实现全局生效
- 避免修改业务逻辑、组件结构、导出逻辑与非主题相关样式

### 范围边界
**包含：**
- 顶部栏主色
- 侧边栏深色背景
- Element Plus 主按钮/聚焦色
- 表格头、矩阵 hover、高亮、弹窗头部渐变等依赖主题变量的区域

**不包含：**
- 页面布局调整
- 字体、阴影、圆角、间距调整
- 组件重构
- 深色模式之外的额外主题系统设计

### 验收标准
- 主页面不再呈现紫色主视觉，改为统一浅蓝色风格
- 按钮、表格、矩阵、弹窗等主题相关区域风格一致
- 文本可读性与 hover/active 状态保持清晰
- 改动尽量集中在单文件，避免扩大 diff

## 已确认上下文
- 主题色来源集中在 `frontend/src/styles/main.css`
- 原子色板与语义变量定义位于 `frontend/src/styles/main.css:2-39`
- Element Plus 主题覆盖位于 `frontend/src/styles/main.css:66-79`
- `frontend/src/main.js:1-17` 仅负责引入 `./styles/main.css`
- `frontend/src/App.vue:801-809` 的弹窗头部渐变使用 `var(--color-primary-subtle)`，无需单独改色值

## 最小改动文件清单
1. `frontend/src/styles/main.css`

## 推荐浅蓝色方案
- 主色：`#0ea5e9`
- hover：`#0284c7`
- 浅色：`#e0f2fe`
- subtle：`#f0f9ff`
- 深色侧边栏：`#0c4a6e`
- 暗色模式主色：`#38bdf8`
- 暗色模式深色背景：`#164e63`

## 实施步骤
1. 修改 `frontend/src/styles/main.css` 中原子色板的紫色/靛蓝色值为浅蓝色系
2. 同步调整 `--color-primary-subtle` 与 Element Plus 相关浅色层级变量
3. 保持语义变量名称不变，确保现有调用点自动继承新颜色
4. 最小验证主页面关键区域显示效果

## 验证要点
- 顶部栏背景是否为浅蓝色
- 侧边栏背景是否仍具备足够对比度
- 主按钮与 hover 状态是否正常
- 表格头部、矩阵 hover、选中态是否一致
- 弹窗头部渐变是否自然过渡为浅蓝色系

## 风险与注意事项
- 过浅的蓝色可能导致浅背景区域对比度不足，应优先保证标题文字与交互状态清晰
- 由于样式依赖语义变量传播，原则上无需改动组件文件；若出现局部残留紫色，再做精确补点
